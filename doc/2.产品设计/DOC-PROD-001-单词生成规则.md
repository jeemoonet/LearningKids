# DOC-PROD-001 单词生成规则

> 目标用户：英语基础较弱的初中生（初一～初三），以北京中考英语备考为主。  
> 相关实现：`scripts/build_vocabulary_db.py`、`src/modules/vocab-training/wordForms.ts`  
> 词表与真题来源：`material/english/单词表.md`、`material/english/*.md`  
> **术语定义**：[DOC-PROD-002 术语表](DOC-PROD-002-术语表.md)（年级 / 场景 / 主题 / 单词字段）

## 1. 设计目标

单词数据（释义、例句、关联词）应服务于**「一眼能懂、能读出声、能用在考试里」**，而不是追求词典式的信息完整。

| 维度 | 原则 |
|------|------|
| 可读性 | 除目标词外，句中其余词尽量为中考高频词 |
| 可理解性 | 一词多义时，每条例句只体现**一个**最常考含义 |
| 可迁移性 | 例句与关联词优先对接真题场景与固定搭配 |
| 低负担 | 弱基础学生不宜一次展示过多英文信息 |

## 2. 适用范围

| 项目 | 说明 |
|------|------|
| 词库 | `words` 表，约 1645 词，分**初级 / 中级 / 高级**三个**年级** |
| 词表 | `material/english/单词表.md`（含词性、考试频率标注） |
| 展示页面 | 记忆、闪卡、测试、快速初始化 |
| 频率字段 | `freq_label`（高频/中频/低频）、`exam_year_count`、`exam_total_count` |

频率标注**仅写入独立字段**，不得拼入 `word`、例句或关联词文本。

**熟词池导出**（编写例句/短文时，除目标词外优先从下列文件选词）：

| 文件 | 说明 |
|------|------|
| `server/data/中考高频词-初级组.md` | 初级年级 175 词 |
| `server/data/中考高频词-中级组.md` | 中级年级 218 词 |
| `server/data/中考高频词-高级组.md` | 高级年级 152 词 |

格式：纯单词，逗号分隔，末尾分号（如 `front, stress, advice;`）。重新导出：`python scripts/export_high_freq_words.py`

## 3. 例句规则

### 3.1 核心原则

| 原则 | 要求 |
|------|------|
| **短** | 6～10 个英文词为宜，最多不超过 12 个 |
| **熟** | 除目标词外，其余词优先使用中考高频词 |
| **真** | 优先使用真题/教材原句；没有再使用人工模板 |
| **一义一句** | 多义词每条例句只体现一个最常考义项 |
| **必有中文** | 每条例句必须提供中文对照 |

### 3.2 例句来源优先级

```text
1. 真题原句（该词在 material/english 试卷 MD 中实际出现过的句子）
2. 教材 / 课标例句
3. 按本节规则人工编写的简单句
4. 禁止使用：通用模板 + 随机填词（如 Students {word} new skills）
```

高频词（`freq_label = 高频`）应优先匹配真题例句，并与 `(N届/M次)` 频率数据联动标注。

### 3.3 句式分级（按年级）

#### 初级年级（beginner）

- 句式：主谓宾 / 主系表，一般现在时为主
- 禁止：从句、被动语态、虚拟语气
- 目标词位置：尽量在句末或谓语位置，便于定位

示例：

```text
I like music.
My mother is kind.
We go to school every day.
```

#### 中级年级（intermediate）

- 允许：1 个介词短语或 1 个并列结构
- 仍避免：`which/that` 定语从句

示例：

```text
After class, we often play basketball.
Tom wants to be a doctor when he grows up.
```

#### 高级年级（advanced）

- 允许：中考阅读 / 完形常见句型（`It is ... to ...`、`help sb. do` 等）
- 约束：句中生词（对该组而言）不超过 1 个

示例：

```text
Reading helps us learn more about the world.
It is important for us to protect the environment.
```

### 3.4 例句情境优先级

编写**例句**时，选材情境按以下顺序选取（与内容结构中的「场景」——学习/家庭/个人——不同，此处指例句发生的背景偏好）：

1. **校园**：上课、作业、同学、老师、考试
2. **家庭**：父母、家务、周末
3. **日常**：吃饭、运动、购物、天气、交通
4. **情感态度**：happy / proud / worried 等

### 3.5 各页面例句展示数量

| 页面 | 例句数量 | 说明 |
|------|----------|------|
| 记忆 | 0～1 句 | 可降低负担，侧重发音 + 释义 |
| 闪卡 | 1～2 句 | 建议 1 句真题 + 1 句简单造句 |
| 测试 | 0 句 | 只考「词 → 义」，不出完整例句 |
| 快速初始化 | 不展示 | 仅展示单词、频率图标、释义 |

**禁止**为凑数固定展示 3 句，尤其禁止展示无中文的模板句。

### 3.6 禁止项

- 把词硬塞进语法不通的模板句
- 学术化表述（如 `plays an important role in this story`）
- 超纲词堆叠（同屏出现多个学生未学词汇）
- 语法错误（介词、时态、及物/不及物混用）
- 将考试频率标注写入例句正文

## 4. 关联单词规则

前端展示标题为「关联单词」；数据按以下**类型优先级**组织，**不得**将 WordNet 同义词直接当作「同词根」展示。

### 4.1 类型 A：词形变化（最优先，必展示）

| 词性 | 关联内容 |
|------|----------|
| 动词 | 过去式 / 过去分词 / 现在分词（不规则动词必记） |
| 名词 | 复数形式 |
| 形容词 / 副词 | 比较级、最高级 |

示例：`take → took → taken → taking`

> 词形变化可在前端通过 `wordForms.ts` 规则推导；数据库侧可逐步补充 `word_forms` 字段。

### 4.2 类型 B：固定搭配（第二优先）

- 每条搭配配一个超短例句
- 每个词只保留中考常考的 **1～2 条**搭配，不列词典式长列表

示例：

```text
look → look at / look for / look after
make → make friends / make progress
```

### 4.3 类型 C：同主题词（第三优先）

同一**主题**或**场景**下的词，而非词典同义词：

```text
school → classroom, teacher, homework, exam
happy → sad, excited, worried（情感对照）
```

### 4.4 类型 D：易混词（第四优先）

适合测试与辨析，用「对比 + 一句说明」即可：

```text
look / see / watch
borrow / lend
```

### 4.5 不建议提供的内容

- WordNet 生僻同义词（如 `can` → `tin`）
- 词根词缀推导（`-tion`、`-ment` 等）——留到中级年级以后
- 一次展示 3 个以上无分类的关联词

### 4.6 各年级展示数量

| 年级 | 关联词展示 |
|------|------------|
| 初级 | 词形变化 **或** 1 组固定搭配（二选一为主） |
| 中级 | 词形变化 + 1 组固定搭配 |
| 高级 | 词形变化 + 搭配 + 1 个易混词 |

## 5. 释义规则（简要）

| 规则 | 说明 |
|------|------|
| 语言 | 中文，面向初中生，避免学术词典体 |
| 义项 | 默认只展示**最常考的一个**义项；多义项在高级年级可扩展 |
| 来源 | 优先人工校对的简明释义；机器翻译结果需审核 |
| 长度 | 一般不超过 20 个汉字 |

---

## 6. 推荐数据结构（后续迭代）

当前 `similar1/2/3` 混装同义词，建议逐步拆分为：

| 字段 | 内容 |
|------|------|
| `example_en` / `example_zh` | 主例句（真题或简单句） |
| `example_en_2` / `example_zh_2` | 可选第二例句 |
| `collocation` | 固定搭配，如 `look after` |
| `confusable_with` | 易混词，如 `borrow` |
| `word_forms` | JSON：`{ past, pp, plural, comparative }` |
| `related_theme_words` | 同场景 / 同主题词，最多 2 个 |
| `freq_label` 等 | 频率独立字段，与正文分离 |

前端「关联单词」按类型分块展示，优于单行 `word1 · word2 · word3`。

## 7. 与现有实现的对照

| 现状 | 问题 | 目标 |
|------|------|------|
| `EXAMPLE_TEMPLATES` 随机填词 | 常产生语法错误句 | 真题优先 + 分级人工句 |
| `get_similar_words()` 使用 WordNet | 同义词过难，且与 UI「同词根」不符 | 词形 / 搭配 / 主题 / 易混 |
| 闪卡固定 3 例句 | 2 句无中文、信息过载 | 按页面控制数量（见 3.5） |
| 频率曾拼入 `word` 字段 | 污染全库 | 仅写 `freq_*` 字段（已清洗） |

## 8. 质量检查清单

### 例句

- [ ] 长度 ≤ 12 词，语法正确
- [ ] 除目标词外无生僻词堆叠
- [ ] 有中文对照
- [ ] 高频词优先使用真题句
- [ ] 不含 `[高频]` 等频率标注

### 关联词

- [ ] 分类明确（词形 / 搭配 / 主题 / 易混）
- [ ] 无 WordNet 生僻同义词充数
- [ ] 数量符合所属年级上限（见 4.6）
- [ ] 拼写与卡片单词原形一致

### 释义

- [ ] 中文简洁，单义项为主
- [ ] 与例句语境一致
