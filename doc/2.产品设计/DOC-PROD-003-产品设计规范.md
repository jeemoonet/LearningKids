# DOC-PROD-003 产品设计规范

> 本规范约束 LearningKids **产品界面与交互**的共性原则，供产品设计、前端实现与 Agent 协作时引用。  
> 术语见 [DOC-PROD-002 术语表](DOC-PROD-002-术语表.md)。

## 1. 适用范围

- 用户端 Web（`src/` 下各学习模块、首页）
- 管理后台（`src/admin/`）同样遵循宽屏优先，可单独细化表格与表单列宽

## 2. UI 设计原则

### 2.1 PC 宽屏优先（核心）

**LearningKids 的 UI 面向 PC 电脑、宽屏环境设计与验收**，不以手机竖屏为默认目标。

| 维度 | 要求 |
|------|------|
| 目标设备 | 桌面/笔记本浏览器，宽屏（内容区建议有效宽度 ≥1280px） |
| 布局 | 横向分栏、多列网格、并排面板；避免在宽屏下仍单列窄条居中 |
| 信息密度 | 进度、词表、操作按钮可在同一屏可见；减少无必要的全屏纵向滚动 |
| 交互 | 鼠标为主（点击、悬停）；按钮与可点区域尺寸适中，无需按移动端拇指热区放大 |
| 断点 | 不强制做移动端适配；若窗口变窄，允许横向滚动或自然折行，但不以窄屏重新设计信息架构 |

### 2.2 布局参考

```text
┌──────────────────────────────────────────────────────────────┐
│  模块头（标题 / 说明 / 返回）                                    │
├─────────────────────────────┬────────────────────────────────┤
│  主内容区（列表 / 卡片 / 练习）   │  侧栏（进度 / 说明 / 次要操作）      │
│  可多列网格                      │  可选，宽屏时并列展示                │
└─────────────────────────────┴────────────────────────────────┘
```

- 首页模块入口：宽屏下多列卡片网格（参考现有 `home-entries`）
- 单词/自由背单词：词表、筛选、统计横排；单页 `max-width` 可设上限（如 920–1200px）但**在宽屏内居中展开**，而非固定手机宽度

### 2.3 文案与语言

- 用户可见文案：**简体中文**
- 日期、数字格式：中文习惯（与项目规则一致）

### 2.4 视觉一致性

- 沿用现有模块样式前缀（如 `module-*`、`vocab-*`、`fv-*`），新模块新增独立前缀，避免污染全局
- 主色与圆角、间距与 `App.css` 中已有模块保持一致；不引入与全站风格冲突的新设计系统

## 4. 验收自检（新页面/改版）

- [ ] 在 1440×900 或更宽窗口下打开，主流程无需横向挤压成窄条
- [ ] 关键操作（下一步、确认、返回）在首屏或固定底栏可见
- [ ] 列表/卡片在宽屏下利用多列或分栏，而非单列占满高度
- [ ] 文案为简体中文，术语与 DOC-PROD-002 一致
